# QuizRoom

Учебное веб-приложение для проведения квизов в реальном времени. Организатор создаёт вопросы и запускает комнату, участники подключаются по шестизначному коду, отвечают во время показа вопроса и видят итоговый лидерборд.

[Макет интерфейса в Figma](https://www.figma.com/design/KnDZ2UGtFCn6y4IaXVBN7F?node-id=1-2)

## Возможности

- регистрация и авторизация с ролями «Организатор» и «Участник»;
- создание и настройка квизов;
- вопросы с одиночным и множественным выбором;
- изображения в вопросах;
- игровые комнаты с кодом подключения;
- QR-код и гостевой вход без регистрации;
- синхронная демонстрация вопросов через Socket.IO;
- автоматическое восстановление комнаты после обновления страницы;
- таймер и блокировка ответов после окончания времени;
- пояснения правильных ответов после завершения раунда;
- звуковые сигналы с возможностью отключения;
- начисление баллов с небольшим бонусом за скорость;
- промежуточный и итоговый лидерборд со статистикой точности и времени ответа;
- история проведённых квизов и участий;
- адаптивный интерфейс для компьютера и телефона.

## Технологии

- React 19 и Vite — клиентский интерфейс;
- Node.js и Express — REST API и раздача готового приложения;
- Socket.IO — обмен событиями в реальном времени;
- SQLite и better-sqlite3 — локальная база данных;
- JWT и bcrypt — авторизация и хранение паролей;
- QRCode и Web Audio API — локальная генерация QR-кодов и звуков без внешних сервисов;
- Vitest, Supertest и Socket.IO smoke-test — автоматические проверки.

## Быстрый запуск

Требуются Node.js 20+ и pnpm.

```bash
pnpm install
pnpm seed
pnpm dev
```

В Windows PowerShell при запрете выполнения `.ps1` используйте `pnpm.cmd` вместо `pnpm`.

После запуска:

- интерфейс: `http://localhost:5173`;
- сервер: `http://localhost:3001`.

Демонстрационные пользователи:

| Роль | Email | Пароль |
|---|---|---|
| Организатор | `organizer@quizroom.local` | `quiz123` |
| Участник | `player@quizroom.local` | `quiz123` |

Чтобы проверить игру, откройте приложение в двух окнах: войдите организатором и участником, запустите демонстрационный квиз и введите код комнаты во втором окне.

## Сборка для запуска одним сервером

```bash
pnpm build
pnpm start
```

Готовое приложение откроется на `http://localhost:3001`.

## Проверки

```bash
pnpm test
pnpm test:smoke
```

Для smoke-теста сервер должен быть запущен. Тест создаёт комнату, подключает участника, отправляет правильный ответ, восстанавливает подключение и проверяет итоговый рейтинг со статистикой.

## Структура

```text
client/              React-интерфейс
  src/App.jsx        основные экраны и игровые сценарии
  src/sounds.js      звуковые сигналы через Web Audio API
server/
  app.js             REST API
  realtime.js        события Socket.IO и игровая логика
  auth.js            JWT-авторизация
  db.js              схема и запросы SQLite
  seed.js            демонстрационные данные
docs/                архитектура и материалы для сдачи
uploads/             изображения вопросов
data/                локальная база SQLite
```

## Учебные ограничения

Проект рассчитан на один сервер и не использует Redis для синхронизации нескольких экземпляров приложения. Активные таймеры комнат хранятся в памяти и не восстанавливаются после перезапуска сервера. Для учебного MVP это упрощает запуск, но в промышленной версии потребовались бы внешнее хранилище состояния, объектное хранилище изображений и расширенное нагрузочное тестирование.
